# Nmap Network Reconnaissance Lab

## Objective

Build a virtual penetration testing lab using Ubuntu Server and perform
network reconnaissance using Nmap.

## Reason behind this project

The purpose of this project was to stregthen my knowledge and understanding of networking, Linux fundamentals
and to demonstrate problem-solving, troubleshooting and critical thinking skills as well as the knowledge of
different network protocols, the usage of Nmap and the ability to document the whole process.


## Technologies

- Ubuntu Server
- VirtualBox
- Nmap
- OpenSSH
- Windows 11

## Skills Demonstrated

- Linux Administration
- Virtualization
- Networking
- Port Scanning
- Service Enumeration
- Troubleshooting
- Network Troubleshooting
- Technical Documentation
- Security Assessment Methodology

## Methodology

1. Configure Ubuntu Server
2. Configure Bridged Networking
3. Verify Connectivity
4. Install OpenSSH
5. Perform TCP Connect Scan
6. Perform Version Detection
7. Enumerate SSH Service
8. Analyze Findings

## Findings

Open Port:
22/TCP

Service:
OpenSSH 10.2p1

Risk:
No obvious vulnerabilities identified during the assessment.

## What I learned
- NAT and Bridged networking.
- Troubleshooting connectivity issues before assessments
- How TCP Connect scans differ from default host discovery.
- An open port represents an attack surface, not automatically a vulnerability.
- The importance of service enumeration before assessing security risks.
- Improved technical documentation and reporting skills.
